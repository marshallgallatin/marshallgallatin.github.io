I definitely learned something new this week.  Since when did the C++ standard include Lambdas?  This is awesome.  Now all that Haskell I learned this summer seems to be a lot more useful.  Its amazing how different programming languages allow you to think about problems in different ways.  A few years ago, I never would have been excited about lambda expressions.

I also am a huge fan of the new standard library function pointer declarations.  The syntax `function<int (int)>` is SOOOOO much more intuitive than the old style.  I always had to lookup the old style since its so strange in comparison to most of the rest of C++ syntax.

I'm not sure about the auto keyword.  It appears to mostly be some syntactic sugar which makes programming easier when types are well known, but also when misused could be misleading.  I guess its like anything, misusing it could have unanticipated consequences.  I'm definitely going to start trying to use it inside for loops, thats one place where typing `container<type>::iterator` always seemed like overkill.

We haven't had a project due in a while, mostly because we have a test this week.  Honestly, I'm a little nervous about the test.  I'm confident in my C++ abilities, but getting formally tested over it is still a little nerve wracking.  I don't do a ton of generic programming, but I have written a few templated classes before, so I'm feeling fairly confident.

## Tip of the week ##
The website [ideone.com](http://ideone.com/) is really cool.  You can write code, run the code, and share the code.  You can send links to others, they can then fork it, edit it, and send it back to you.  I guess its a great collaboration tool - its really handy for figuring out if something works, or showing someone else exactly what you mean.  It also supports numerous languages, so its handy for everything from JavaScript to Scheme.
